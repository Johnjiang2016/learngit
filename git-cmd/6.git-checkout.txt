1、 改错了工作区域的文件，丢掉工作区域的文件：
    git checkout -- <file>
    一种是文件自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
    一种是文件已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。
    总之，就是让这个文件回到最近一次git commit或git add时的状态。
2、改错了工作区域的文件，并且添加到了暂存区域，想丢掉暂存区域的文件
    git reset HEAD <file> 这时候退回到 步骤1

3、（没有推送到远程仓库）已经提交了不合适的修改到版本库时，想要撤销本次提交，只能通过 文件 git-reset.txt 回退。

4、推送到远程仓库，首先回退到 正确的版本，比如回退到上一个版本
    git reset --hard HEAD^
    然后删除远程分支
    git push origin --delete <BranchName>
    接着把本地分支推送到远程仓库
    git push origin <BranchName>
